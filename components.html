Accessible Search Bar

Recreating Nintendo's search bar for inclusive interaction across all inputs.

The search bar is a deceptively simple component. It’s ubiquitous, yet surprisingly inconsistent across platforms when it comes to accessibility. I chose to redesign the search bar component using Nintendo's implementation as a model, with accessibility at the forefront.

Input Accessibility

  • Nintendo: Clean input, clear focus style, full keyboard accessibility
  • Amazon: Extensive autocomplete, but keyboard interaction is noisy
  • Suruga-Ya: Lacks visual feedback and has inconsistent label structure

I compared mouse, keyboard, and mobile interactions across all three and recorded how users can activate, edit, and submit text. The most accessible component was Nintendo’s—each action was predictable and screen reader-friendly.

Output Accessibility

Nintendo's search bar provides strong feedback: focus outlines, placeholder text, and a visible clear button. Suruga-Ya offers none of these. Screen reader support was also minimal on Suruga-Ya compared to Amazon and Nintendo.

Focus order on Nintendo made sense, but Suruga-Ya skipped elements or reordered them, which disrupted keyboard-only navigation.

State Models

I diagrammed separate state models for mouse and keyboard usage. For instance, the focused state is reached via click or tab key. Autocomplete state activates after 2+ characters typed. Disabled or empty states have distinct visual and screen reader output.

I then revised these state models to optimize clarity for keyboard users and screen readers—for example, allowing ESC to exit autocomplete mode cleanly, or improving how placeholder text is described.

My Redesign

Final redesigned component using Figma.

  • Announces placeholder and input mode to screen readers
  • Has robust keyboard-only support (e.g., arrow keys in suggestions)
  • Maintains visual focus ring with high contrast

Reflection

Designing an accessible component forced me to think in terms of edge cases and non-mouse users. I was inspired by Nintendo’s clarity and disappointed by Suruga-Ya’s ambiguity.

The biggest challenge was balancing minimalism with usability—for example, having a visible label might clutter the page, but it's crucial for screen readers. I now recognize that accessibility isn’t an add-on—it’s a core design decision.
