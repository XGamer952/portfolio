Partiful Itineraries

Designing a smooth interface for multipart events through iterative prototyping

Partiful is a popular platform for hosting and RSVPing to events, especially parties. But one major limitation was its inability to support multi-part events—things like bar crawls, day trips, or weekend getaways with multiple activities.

This project tackled the design of a flexible itinerary feature that let hosts create and manage multipart events intuitively—while still feeling as seamless and delightful as Partiful’s brand promises.

Goal: Design a flexible, fun, and frictionless interface for planning events with multiple sub-events, without overwhelming either hosts or guests.

Approach

We used a human-centered iterative design process, involving interviews, storyboarding, sketching, wireframing, testing, and refinement. We worked in a team of two and constantly checked designs against user needs and Partiful’s voice.

  • User Research: Interviews with active Partiful users and event planners
  • Storyboarding: Key user flows and edge case planning
  • Prototyping: Multiple iterations of low- and hi-fi mockups
  • Feedback: Sessions with PMs, designers, and the Partiful team

Sketches

Early ideation explored RSVP logic, mobile layouts, subevent privacy, and creation flows.

We asked ourselves: Should guests RSVP to each part of an event? Can hosts easily create subevents on mobile? How should private or invite-only subevents be displayed?

Wireframes

We built low-fidelity wireframes in Balsamiq and ran testing sessions to validate flows.

  • One-page vs multi-page subevent creation
  • Per-event RSVP toggles and default RSVP-all option
  • Comment and photo feeds scoped to each subevent

Feedback

Throughout the process, we sought critique from industry designers and iterated fast.

  • "Let RSVP be simple by default, with more control tucked away for advanced users"
  • "Avoid making each subevent feel like a standalone page—make it feel like a scrollable story"
  • "Keep the party vibe. Don’t overengineer it."

Hi-Fi Prototype

The final design balances fun and functionality. Hosts can add parts easily. Guests can RSVP per part or all at once.

  • Timeline-style scroll with anchored headers per subevent
  • Mobile-first layout with responsive design principles
  • Clear RSVP indicators and editable privacy settings

Conclusion

We designed a scalable itinerary feature for Partiful that maintains the product’s magic while solving real host frustrations. The flexible structure supports everything from chill dinners to multi-day festivals.

Next steps: Test with live users, refine edge cases, add RSVP summaries, and explore poll/map integration.
